# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** GitHub Actions gate (backend + frontend + e2e) on every PR/push, plus deploy-to-Railway with a live smoke check on merges to main.

**Architecture:** One workflow file with four jobs; a small conftest change makes Postgres-unreachable a hard failure under `CI`; the frontend job mechanically enforces the committed-template policy; deploy runs `railway up` via a `RAILWAY_TOKEN` secret only after all gates pass.

**Tech Stack:** GitHub Actions (ubuntu-latest), Postgres 16 service container, Node 20, Python 3.13, docker compose on the runner, Playwright, Railway CLI.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-12-ci-cd-design.md`.
- Backend CI runs Python **3.13** (production-image parity). Local dev stays 3.14 — the conftest change must not alter local behavior (skip when Postgres down locally; FAIL when `CI` env set).
- Frontend job must fail if `overturn/templates/workbench.html` is out of sync with `npm run build:template` output.
- Deploy job: main pushes only, `needs` all three test jobs, auth solely via the `RAILWAY_TOKEN` repo secret (the owner creates it; it never appears in the repo or chat), bounded smoke check that fails loudly.
- Concurrency cancel-in-progress; explicit `timeout-minutes` on every job.
- Task 3 is CONTROLLER-EXECUTED (it needs the owner for the secret and the merge); implementer subagents handle Tasks 1–2 only.
- Every commit message ends with: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`

- [ ] **Step 1: Make the Postgres-unreachable path CI-aware (with a local test)**

In `tests/server/conftest.py`, the `engine` fixture currently does:

```python
    except OperationalError:
        pytest.skip(
            "Postgres unreachable — start it with `docker compose up -d db`",
            allow_module_level=False,
        )
```

Replace with:

```python
    except OperationalError:
        message = "Postgres unreachable — start it with `docker compose up -d db`"
        if os.environ.get("CI"):
            pytest.fail(f"{message} (refusing to skip in CI)", pytrace=False)
        pytest.skip(message, allow_module_level=False)
```

(`import os` at the top of the file if not present.)

Verify both behaviors locally:

Run: `TEST_DATABASE_URL=postgresql+psycopg://nobody:nowhere@localhost:1/none .venv/bin/python -m pytest tests/server/test_models.py -q`
Expected: `skipped` results (local behavior unchanged).

Run: `CI=1 TEST_DATABASE_URL=postgresql+psycopg://nobody:nowhere@localhost:1/none .venv/bin/python -m pytest tests/server/test_models.py -q 2>&1 | tail -2`
Expected: FAILED with "refusing to skip in CI".

Run: `.venv/bin/python -m pytest tests/ -q`
Expected: 155 passed (real DB up, nothing changed).

- [ ] **Step 2: Write the workflow (backend + frontend jobs)**

`.github/workflows/ci.yml`:

```yaml
name: ci

on:
  pull_request:
  push:
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  backend:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: overturn
          POSTGRES_PASSWORD: overturn
          POSTGRES_DB: overturn
        ports:
          - 5432:5432
        options: >-
          --health-cmd "pg_isready -U overturn"
          --health-interval 5s
          --health-timeout 5s
          --health-retries 10
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.13"
          cache: pip
      - name: Create test database
        run: |
          PGPASSWORD=overturn psql -h localhost -U overturn -d overturn \
            -c "CREATE DATABASE overturn_test OWNER overturn"
      - name: Install
        run: pip install -e ".[dev,server]"
      - name: Pytest
        env:
          TEST_DATABASE_URL: postgresql+psycopg://overturn:overturn@localhost:5432/overturn_test
        run: pytest tests/ -q

  frontend:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: frontend/package-lock.json
      - name: Install
        working-directory: frontend
        run: npm ci
      - name: Vitest
        working-directory: frontend
        run: npm test
      - name: Build SPA
        working-directory: frontend
        run: npm run build:app
      - name: Template in sync with source
        working-directory: frontend
        run: |
          npm run build:template
          git diff --exit-code ../overturn/templates/workbench.html \
            || (echo "::error::overturn/templates/workbench.html is out of sync — run 'npm run build:template' and commit it" && exit 1)
```

Validate the YAML parses:

Run: `.venv/bin/python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/ci.yml')); print('yaml ok')"`
(If PyYAML isn't installed in the venv, `python3 -c` with the system
Python or `pip install pyyaml` into the venv — the check matters, not the
interpreter.)
Expected: `yaml ok`

- [ ] **Step 1: Append the e2e job**

Append to the `jobs:` map in `.github/workflows/ci.yml`:

```yaml
  e2e:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    steps:
      - uses: actions/checkout@v4
      - name: Start stack
        run: docker compose up -d --build
      - name: Wait for app
        run: |
          for i in $(seq 1 60); do
            code=$(curl -s -o /dev/null -w '%{http_code}' http://localhost:8000/api/v1/demo/claims || true)
            [ "$code" = "200" ] && exit 0
            sleep 5
          done
          echo "::error::app never became healthy"
          docker compose logs --tail 100
          exit 1
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: frontend/package-lock.json
      - name: Install frontend deps + browser
        working-directory: frontend
        run: |
          npm ci
          npx playwright install --with-deps chromium
      - name: Playwright e2e
        working-directory: frontend
        run: npm run e2e
      - name: Upload report on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: frontend/playwright-report/
          if-no-files-found: ignore
```

- [ ] **Step 2: Append the deploy job**

```yaml
  deploy:
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    needs: [backend, frontend, e2e]
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4
      - name: Install Railway CLI
        run: npm i -g @railway/cli
      - name: Deploy web
        env:
          RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}
        run: railway up --service web --detach
      - name: Deploy worker
        env:
          RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}
        run: railway up --service worker --detach
      - name: Smoke check
        run: |
          for i in $(seq 1 60); do
            spa=$(curl -s -o /dev/null -w '%{http_code}' https://overturn.up.railway.app/ || true)
            api=$(curl -s -o /dev/null -w '%{http_code}' https://overturn.up.railway.app/api/v1/demo/claims || true)
            [ "$spa" = "200" ] && [ "$api" = "200" ] && echo "live" && exit 0
            sleep 10
          done
          echo "::error::smoke check failed — site not healthy after deploy"
          exit 1
```

Re-validate YAML (same command as Task 1).

### Task 3 (CONTROLLER-EXECUTED): Prove the pipeline live

No files. This task exercises the pipeline for real and involves the owner.

- [ ] **Step 1:** Push the branch, open a PR ("CI/CD: test gate + deploy on merge"). The PR run IS the test: all three jobs (`backend`, `frontend`, `e2e`) must go green (`gh pr checks --watch`). Expect the first run to be the slowest (no caches).
- [ ] **Step 2 (negative test):** Push a scratch commit to the PR branch with a deliberately failing assertion (e.g. append a failing test to `tests/server/test_models.py`), confirm the `backend` job goes red and that `deploy` is not eligible, then revert the commit and confirm green again. This proves the gate gates.
- [ ] **Step 3 (owner action):** Ask the owner to create a Railway project token (dashboard → project `overturn` → Settings → Tokens) and store it: `gh secret set RAILWAY_TOKEN` (typed by the owner, e.g. via `! gh secret set RAILWAY_TOKEN` in-session so it never passes through the assistant). Verify existence with `gh secret list` (name only).
- [ ] **Step 4:** Owner merges the PR. Watch the main push run: all gates + `deploy` green; verify the smoke check passed and independently probe the live site. Recommend (owner, optional): branch protection requiring the three checks.
- [ ] **Step 5:** Ledger + Linear sync; delete the branch.
